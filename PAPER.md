# PAPER.md

## Working Title

quantum_bench: A Multi-Agent LLM Experiment in Applied Quantum Mechanics Computation

## Primary Contribution

The LLM experiment and error taxonomy -- not the physics results.

## Central Research Question

Can a multi-agent LLM system (Claude as architect, Codex as coder) implement correct
numerical solutions to applied quantum mechanics problems, verified against pure Ruby
benchmark computations derived from first principles?

## Experiment Design

- Test vehicle: Tier 2 applied quantum mechanics problems
- Benchmark: pure Ruby implementations using Numo::NArray
- LLM implementations: generated by Codex from Claude-designed prompts
- Evaluation: numerical comparison, pass/fail, error classification
- Error taxonomy: wrong_theorem, wrong_boundary, normalization_failure,
  arithmetic_error, hallucinated_step, correct

## Series Context

This is the third paper in a series on multi-agent LLM scientific software development:

1. stellar_pop -- astrophysics, loop not closed (diffuse ground truth)
2. protein_variants -- biology, loop closed but problem too trivial
3. quantum_bench -- applied quantum mechanics, Goldilocks zone target
